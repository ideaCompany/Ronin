As simple as possible

We believe that in the complex world of software development simplicity is key! The best way to keep a clear overview of these complex systems is through keeping every single step in the process as simple and clear as possible. Even when it comes to technical analysis and architecture we still put a big focus on trying to keep complex systems as simple as possible. Most projects may start out smooth, but the real art of software engineering is keeping it that way. Not a small feat, but one that can be accomplished through simplicity, automated testing and SOLID design principles, the three pillars on which we develop our software.

Test Drivin Design

Using the Test Driven Design (TDD) brings with it a number of significant advantages. TDD is the fastest way to develop software because testing the software while it is being built gives such fast feedback it drastically reduces time spent on rework. This means less time spent in the debugger because you will be able to identify the error or problem more quickly. This effectively increases the programmer's productivity and drastically shortens the development Time to Market. We will instantly be able to tell whether the last change has broken previously working code. It also forces radical simplification of the code, we will only write code in response to the requirements of the tests. TDD creates code based on the SOLID principles which allows the design to evolve and adapt to our changing understanding of the problems and will make everything maintainable, flexible and easily extensible. In short, TDD is the fastest, most adaptable way to creating software of the highest quality.

Iterative And Incremental

Our workflow will follow a process which is iterative and incremental. This means that instead of going through on big development cycle which consists of one planning phase, one building phase, one testing phase and one review phase before deployment we will go through multiple cycles of these phases. We divide the software up into sections for each of which we can plan, build, test and review. Then we keep repeating this process in a cyclical manner until we get the desired outcome. This is what it means to work in an iterative manner, think of it as working in short cycles. Because we work on separate sections at the same time and in a cyclical manner we will improve the overall project one step at a time, one section at a time. This means the software will improve incrementally. This allows us to implement software one section at a time while ensuring it's correct integration with previously implemented sections of software.
